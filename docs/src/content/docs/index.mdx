# Tài liệu Calorie Web

Chào mừng đến với tài liệu dự án **Calorie-Web**. Đây là một ứng dụng web hiện đại được xây dựng bằng Next.js, được thiết kế để giúp người dùng theo dõi và quản lý lượng calo tiêu thụ hàng ngày.

## Cấu Trúc Dự Án

```
calorie-web/
├── app/                    # Chứa các route chính (Main routes and pages)
│   ├── layout.tsx         # Layout gốc của ứng dụng
│   ├── page.tsx           # Trang chủ
│   └── globals.css        # Các style toàn cục
├── components/            # UI components tái sử dụng (Reusable React components)
├── lib/                   # Hàm xử lý logic (Logic handling functions)
│   └── storage.ts        # Quản lý localStorage (Storage management)
├── store/                 # Quản lý trạng thái (State management)
│   └── useAppStore.ts    # Trạng thái toàn cục với Zustand
├── types/                 # Định nghĩa kiểu dữ liệu (TypeScript type definitions)
│   ├── index.ts          # Các interface chính (MacroTarget, UserProfile, ...)
│   └── user.ts           # Các kiểu dữ liệu người dùng cơ bản
├── public/                # Tài nguyên tĩnh (Static assets)
└── docs/                  # Trang tài liệu Starlight
```

## Ứng Dụng Hoạt Động Như Thế Nào

Ứng dụng Calorie-Web tuân theo một quy trình đơn giản và trực quan:

1. **Nhập Dữ Liệu từ Người Dùng** - Người dùng nhập dữ liệu calo thông qua các biểu mẫu
2. **Quản Lý Trạng Thái** - Dữ liệu được lưu trữ trong store Zustand
3. **Cập Nhật UI** - Các thành phần React tự động cập nhật theo thay đổi
4. **Trực Quan Hóa Dữ Liệu** - Biểu đồ hiển thị xu hướng và thống kê calo
5. **Lưu Trữ** - Trạng thái có thể được lưu vào local storage hoặc backend

Quy trình phản ứng này đảm bảo rằng giao diện người dùng luôn đồng bộ với dữ liệu của ứng dụng.

## Quản Lý Trạng Thái

### Tổng Quan Zustand Store

Ứng dụng sử dụng **Zustand** để quản lý trạng thái nhẹ và hiệu quả.

**Những gì được lưu trong `useAppStore.ts`:**
- Thông tin hồ sơ người dùng (tên, tuổi, mục tiêu)
- Bản ghi tiêu thụ calo hàng ngày
- Tùy chọn và cài đặt người dùng
- Mục tiêu và chỉ tiêu calo
- Dữ liệu lịch sử để xem xu hướng

**Tại sao dùng Zustand?**
- Code ngắn gọn hơn so với Redux
- Dễ hiểu và dễ triển khai
- Nhẹ và không có phụ thuộc bên ngoài
- Phù hợp cho các ứng dụng cỡ vừa

**Ví dụ sử dụng:**
```typescript
import { useAppStore } from '@/store/useAppStore';

const MyComponent = () => {
  const { userData, addCalorie } = useAppStore();
  
  return (
    <div>
      <p>Tổng Calo: {userData.totalCalories}</p>
      <button onClick={() => addCalorie(500)}>Thêm 500 calo</button>
    </div>
  );
};
```

## Định Nghĩa Kiểu Dữ Liệu (Type Definitions)

### Tổng Quan

TypeScript là một phần quan trọng của dự án. Tất cả các interface chính được định nghĩa trong `types/index.ts`. Các interface này mô tả cấu trúc dữ liệu và đảm bảo an toàn kiểu dữ liệu (type safety) trong toàn bộ ứng dụng.

### Cấu Trúc Dữ Liệu Chính

| Interface | Mô Tả | Quan Hệ |
|-----------|-------|---------|
| **MacroTarget** | Mục tiêu dinh dưỡng hàng ngày (calo, protein, carbs, fat) | Được sử dụng trong `UserProfile` |
| **UserProfile** | Thông tin cá nhân người dùng và mục tiêu dinh dưỡng | Chứa `MacroTarget` |
| **Ingredient** | Nguyên liệu cơ bản (tên, calo, macros, số lượng) | Được sử dụng trong `MealEntry` |
| **MealEntry** | Một bữa ăn cụ thể (loại, nguyên liệu, thời gian) | Chứa `Ingredient[]`, được sử dụng trong `DailyLog` |
| **DailyLog** | Nhật ký calo toàn bộ một ngày | Chứa `MealEntry[]` |

### Sơ Đồ Mối Quan Hệ (Dependency Diagram)

```
MacroTarget
    ↓
UserProfile

Ingredient
    ↓
MealEntry
    ↓
DailyLog
```

### Ví dụ Sử Dụng

```typescript
import { UserProfile, DailyLog, MealEntry, Ingredient, MacroTarget } from '@/types'

// 1. Tạo MacroTarget
const macroTarget: MacroTarget = {
  calories: 2000,
  protein: 150,
  carbs: 200,
  fat: 65,
}

// 2. Tạo UserProfile
const userProfile: UserProfile = {
  name: 'Nguyễn Văn A',
  age: 25,
  weight: 70,
  height: 175,
  goal: 'lose',
  macroTarget,
}

// 3. Tạo Ingredient
const ingredient: Ingredient = {
  name: 'Gạo trắng',
  calories: 130,
  protein: 2.7,
  carbs: 28,
  fat: 0.3,
  amount: 100,
}

// 4. Tạo MealEntry
const breakfast: MealEntry = {
  id: 'meal-1',
  mealType: 'breakfast',
  time: '08:00',
  ingredients: [ingredient],
  totalCalories: 130,
}

// 5. Tạo DailyLog
const dailyLog: DailyLog = {
  date: '2024-05-08',
  meals: [breakfast],
  totalCalories: 130,
}
```

### Quy Tắc Đặt Tên & Format

- **ID**: Sử dụng string (UUID hoặc custom string)
- **Ngày**: Format `YYYY-MM-DD` (ví dụ: `2024-05-08`)
- **Thời gian**: Format `HH:mm` (ví dụ: `08:00` hoặc `14:30`)
- **Giá trị số**: Calo và macros lưu dưới dạng số (`number`)
- **Goal**: Chỉ chấp nhận 3 giá trị: `lose`, `maintain`, `gain`
- **MealType**: Chỉ chấp nhận 4 loại: `breakfast`, `lunch`, `dinner`, `snack`

## Storage Layer (lib/storage.ts)

### Tổng Quan

Storage Layer (`lib/storage.ts`) cung cấp các hàm an toàn để quản lý localStorage. Nó hoạt động như một lớp trung gian giữa ứng dụng và trình duyệt, đảm bảo:
- ✅ Xử lý lỗi tự động (try-catch)
- ✅ Type-safe với TypeScript
- ✅ Client-only (không chạy trên server)
- ✅ Dữ liệu được validate trước khi lưu/đọc

### Quy Tắc Bắt Buộc

- ❌ **KHÔNG** truy cập `localStorage` trực tiếp từ component
- ✅ **LUÔN** import và sử dụng hàm từ `@/lib/storage`
- ✅ Tất cả lỗi được xử lý tự động

### Các Hàm Chính

| Hàm | Tham số | Trả về | Mô tả |
|-----|---------|--------|-------|
| `getProfile()` | - | `UserProfile \| null` | Đọc thông tin hồ sơ người dùng |
| `saveProfile(profile)` | `profile: UserProfile` | `void` | Lưu thông tin hồ sơ người dùng |
| `getLog(date)` | `date: string (YYYY-MM-DD)` | `DailyLog \| null` | Đọc nhật ký calo theo ngày |
| `saveLog(date, log)` | `date: string, log: DailyLog` | `void` | Lưu nhật ký calo theo ngày |
| `getLogs(startDate, endDate)` | `startDate, endDate: string` | `DailyLog[]` | Lấy logs trong khoảng thời gian |

### Ví dụ Sử Dụng

```typescript
import { 
  getProfile, 
  saveProfile, 
  getLog, 
  saveLog, 
  getLogs 
} from '@/lib/storage'

// 1. Lấy profile hiện tại
const profile = getProfile()

// 2. Lưu profile
saveProfile({
  name: 'Nguyễn Văn A',
  age: 25,
  weight: 70,
  height: 175,
  goal: 'lose',
  macroTarget: {
    calories: 2000,
    protein: 150,
    carbs: 200,
    fat: 65,
  },
})

// 3. Lấy log của một ngày
const todayLog = getLog('2024-05-08')

// 4. Lưu log mới
saveLog('2024-05-08', {
  date: '2024-05-08',
  meals: [],
  totalCalories: 0,
})

// 5. Lấy logs từ khoảng thời gian (7 ngày)
const weekLogs = getLogs('2024-05-01', '2024-05-08')
weekLogs.forEach(log => {
  console.log(`${log.date}: ${log.totalCalories} kcal`)
})
```

### Cơ Chế Lưu Trữ

```
localStorage
├── "calorie_profile"              # Lưu UserProfile
└── "calorie_log_YYYY-MM-DD"       # Lưu DailyLog theo ngày
    ├── "calorie_log_2024-05-08"
    ├── "calorie_log_2024-05-07"
    └── ...
```

### Lưu Ý Quan Trọng

- **Format ngày**: Luôn `YYYY-MM-DD` (ví dụ: `2024-05-08`)
- **Xử lý lỗi**: Hàm trả về `null` hoặc `[]` nếu lỗi, không bao giờ throw
- **Client-only**: Storage chỉ hoạt động trên browser
- **LocalStorage limit**: ~5-10MB, cần quản lý dữ liệu cũ để tránh vượt quá

## Các Thành Phần (Components)

Thư mục `components/` chứa các phần tử UI tái sử dụng. Dưới đây là các thành phần mẫu và vai trò của chúng:

| Tên Thành Phần | Vai Trò | Mục Đích |
|---|---|---|
| `Button` | Điều khiển UI | Nút bấm có các biến thể (primary, secondary) |
| `Input` | Nhập Dữ Liệu | Ô nhập văn bản cho nhập calo và dữ liệu người dùng |
| `Card` | Container | Hiển thị thông tin nhóm (tóm tắt hàng ngày, thống kê) |
| `Chart` | Trực Quan Hóa | Hiển thị xu hướng calo và biểu đồ bằng Recharts |
| `Header` | Điều Hướng | Thanh điều hướng trên cùng với logo và menu |
| `Footer` | Bố Cục | Phần footer với liên kết và bản quyền |

**Cấu Trúc Thành Phần:**
Mỗi thành phần nên:
- Độc lập và tái sử dụng được
- Chấp nhận props để tùy chỉnh
- Bao gồm các kiểu TypeScript
- Tuân theo quy ước đặt tên (PascalCase)

### Quy Ước Mã

#### Quy Ước Đặt Tên

- **Thành Phần:** Sử dụng PascalCase (ví dụ: `UserProfile`, `CalorieInput`)
- **Hàm:** Sử dụng camelCase (ví dụ: `calculateTotalCalories`, `formatDate`)
- **Hằng Số:** Sử dụng UPPER_SNAKE_CASE (ví dụ: `MAX_DAILY_CALORIES`, `API_BASE_URL`)
- **Tệp:** Khớp với tên thành phần (ví dụ: `UserProfile.tsx`)

#### Cấu Trúc Tệp

- Đặt các thành phần trong thư mục `components/`
- Nhóm các thành phần liên quan trong thư mục con nếu cần
- Giữ logic thành phần đơn giản; chuyển logic phức tạp đến `lib/`
- Định nghĩa các kiểu trong thư mục `types/`
- Lưu các hàm tái sử dụng trong thư mục `lib/`

#### Các Thực Hành Tốt Nhất của TypeScript

- Luôn định nghĩa các kiểu cho props và giá trị trả về
- Sử dụng interface cho hình dạng đối tượng
- Tránh kiểu `any`; sử dụng các kiểu cụ thể thay vào đó
- Ví dụ:

```typescript
interface CalorieEntry {
  id: string;
  date: Date;
  amount: number;
  category: string;
}

interface CalorieInputProps {
  onSubmit: (entry: CalorieEntry) => void;
}

const CalorieInput: React.FC<CalorieInputProps> = ({ onSubmit }) => {
  // Triển khai thành phần
};
```

#### Mẹo Phát Triển Thành Phần

1. **Bắt đầu với interface TypeScript** cho props thành phần
2. **Giữ các thành phần nhỏ** - mỗi thành phần nên có một trách nhiệm
3. **Trích xuất logic** vào hàm `lib/` để kiểm tra và tái sử dụng
4. **Sử dụng Zustand hooks** cho trạng thái toàn cục trong thành phần
5. **Tạo kiểu với Tailwind** bằng các lớp tiện ích
6. **Ghi chú logic phức tạp** bằng nhận xét
